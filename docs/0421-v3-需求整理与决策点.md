### B. 题库改造

**你要做的**：
- 打开 `E:\cursor\ProjectsForPrac\coworkersfun\data\questions.json`
- 文件结构（每题 `options` 数组里每个选项 3 个字段）：
  ```json
  {
    "id": "Q1",
    "text": "题干",
    "options": [
      { "label": "选项显示文字", "luck": 1, "tags": ["fire_sign"] }
    ]
  }
  ```
  - `label`：给用户看的字
  - `luck`：-2 到 +2 整数（影响稀有度池；+6 总分给 SSR 加权）
  - `tags`：字符串数组，用来触发**命定卡路径**（当前只有 SSR001 圣诞老人要求 5 个 tag 同时命中）

**我识别到的改题方向**：

| Q | 当前 | 你的评价 | 建议 |
|---|---|---|---|
| Q1 星座 | 12 星座 | 保留（没抱怨） | 不动 |
| Q2 股票 | 4 选项 | 保留（没抱怨） | 不动 |
| Q3 表白 | 4 选项 | ❌ 不合理、难填、不有意思 | **重设计**（参考下方 SBTI 风格） |
| Q4 吃好吃的 | 4 选项 | ❌ 不够有意思 | **重设计** |
| Q5 心情 | 4 选项 | ⚠️ 勉强过得去 | **重设计或删** |
| **Q6 新增** | — | 0-999 数字输入 | **新增一道** |

**SBTI 原版题目风格参考**（我去拉了 `pingfanfan/SBTI` 的 30 道题库看了下）：
- 题干**长而有代入感**，像小说旁白：
  > "我是一只阴暗的老鼠，一只爬行的蟑螂，这辈子没谈过一场恋爱，胆怯又自卑..."
- 题干**设置荒诞场景**：
  > "对象超过 5 小时没回消息，说自己窜稀了，你会怎么想？"
- 题干**加排比铺垫**：
  > "你的恋爱对象是一个尊老爱幼，温柔敦厚，洁身自好，光明磊落，大义凛然..."
- 选项**只要 2 个**（大概同意 / 大概不同意），拉低填题成本

**我的建议（给你改题时参考，不是替你定）**：

```
Q3（原表白题 → 换成）：职场瞬间代入
"老板甩给你一张 excel，里面第一行是'这个数据为什么和上周对不上'。你的第一反应？"
A. 赶紧对着上周的再查一遍  →  tags: ["anxious"]  luck: -1
B. 问老板"你指的是哪列"     →  tags: ["calm"]    luck: +1

Q4（原吃好吃的 → 换成）：工位特写
"你看工位上长期堆了什么？"
A. 一堆奶茶杯               → tags: ["sugar"]    luck: 0
B. 什么都没有，干净得可疑   → tags: ["minimal"]  luck: +1
C. 像考古现场               → tags: ["chaos"]    luck: -1
D. 绿植已经枯了两盆         → tags: ["dead"]     luck: -2

Q5（原心情 → 换成）：出勤画像
"周一早高峰，你是哪款？"
A. 提前 30 分钟到工位，咖啡还没凉  → tags: ["eager"]   luck: +2
B. 挤进会议的前一秒落座              → tags: ["timely"]  luck: 0
C. 打卡 9:02，默默流泪                → tags: ["lazy"]    luck: -1
D. 远程摸鱼（今天休假不说）          → tags: ["slack"]   luck: -2
```

**Q6 数字题（新增） — 我需要你明确玩法**：
- 玩法 a：`{数字 % 48}` 直接固定对应 48 张卡中一张（完全破坏随机）→ **不推荐**（失去抽卡惊喜）
- 玩法 b：数字区间映射 luck +/- 值（如 0-99 → -2；900-999 → +2）→ **推荐（有占卜感）**
- 玩法 c：特殊数字触发彩蛋（比如 250 = 必出某卡）→ 有意思但要定规则
- 玩法 d：数字 → SHA 哈希 → 种子（真随机但有"专属感"，用户觉得"这是我今天的数字")→ **推荐**

**你倾向哪个？** 或者"a + c 组合"之类自定义。

### C. 抽卡机制：1 → 5 张盲盒

**当前**：答完题直接抽 1 张
**v3**：答完题出现 5 张**卡背朝上、悬空**的卡 → 用户**依次点击翻开** → 开宝箱动画

**我需要你明确**：

1. **5 张卡的稀有度组合怎么定？**
   - A. 各自独立按 75/20/5 概率抽 → 可能 5 张全 R 非常平淡
   - B. 至少保证有 1 张非 R（SR 或 SSR）→ 类似 10 连保底
   - C. 按 "3 R + 1 SR + 1 随机" 固定结构 → 结构化保底
   - D. 按"今日运势"：总 luck≥6 → 至少 1 张 SSR

2. **翻开顺序有限制吗？**
   - A. 任意顺序点击 → 自由
   - B. 只能从左到右翻 → 仪式感
   - C. 最后一张总是最高稀有度 → 拉高潮

3. **翻完 5 张后**：
   - A. 定格显示 5 张成一排（供对比、截图、分享）
   - B. 跳转"今日主角"页（从 5 张里选稀有度最高一张放大）
   - C. 进入"五连图鉴"分享海报模式

### D. 翻面特效（按稀有度）

**你的描述**：
- **R**：翻面时一点白色小光
- **SR**：紫色光
- **SSR**：金光 + 特效

**我的技术方案**（你不用决策，供你了解）：
- CSS `transform: rotateY()` + `transform-style: preserve-3d` 做 3D 翻转
- `keyframes` 控制翻转过程中的 box-shadow/filter
- SSR 翻完后用 canvas 或 SVG 粒子喷射（`<canvas-confetti>` 库一行搞定金粉）
- 参考项目：
  - [Mikulew/css-3d-card-rgb-effect](https://github.com/Mikulew/css-3d-card-rgb-effect) — RGB 光效卡片
  - [uusei/get_card](https://github.com/uusei/get_card) — 可自定义卡池抽卡模拟器
  - [pxm95772/YuanShen-MoNiDraw](https://github.com/pxm95772/YuanShen-MoNiDraw) — 原神抽卡前端
